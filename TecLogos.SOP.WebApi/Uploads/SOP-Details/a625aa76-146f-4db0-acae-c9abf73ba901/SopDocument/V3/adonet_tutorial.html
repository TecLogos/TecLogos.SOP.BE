Module 1

Introduction &
ADO.NET Architecture

ADO.NET is Microsoft's data access technology — a set of classes that expose data access services for .NET applications. It provides a bridge between your C# code and a relational database.

What is ADO.NET?

ADO.NET (ActiveX Data Objects for .NET) is part of the .NET Framework / .NET Core that lets you connect to databases, execute SQL commands, retrieve data, and perform CRUD operations.

🔌
Connected Layer
SqlConnection, SqlCommand, SqlDataReader — live database connection required
📦
Disconnected Layer
DataSet, DataTable, DataAdapter — work with data offline, sync later
⚡
High Performance
Forward-only DataReader is extremely fast for read-only scenarios
🔒
Parameterized Queries
Built-in SQL injection prevention using SqlParameter objects

READ — SELECT

SQL — Select Queries

SELECT * FROM Employees;

-- Select specific columns
SELECT EmployeeId, FirstName, LastName, Department
FROM Employees;

-- Filter with WHERE
SELECT * FROM Employees
WHERE Department = 'Engineering' AND Salary > 80000;

-- Sort results
SELECT * FROM Employees
ORDER BY Salary DESC;

-- Aggregate functions
SELECT
    Department,
    COUNT(*) AS HeadCount,
    AVG(Salary) AS AvgSalary,
    MAX(Salary) AS MaxSalary
FROM Employees
GROUP BY Department
HAVING COUNT(*) > 1;

UPDATE — Modify Rows

SQL — Update
-- Update single record
UPDATE Employees
SET Salary = Salary * 1.10,  -- 10% raise
    Department = 'Senior Engineering'
WHERE EmployeeId = 1;

-- Update multiple records
UPDATE Employees
SET Salary = Salary * 1.05
WHERE Department = 'Engineering';

Always use WHERE with UPDATE/DELETE! Omitting WHERE modifies or deletes every row in the table. Always test your WHERE clause with a SELECT first.

DELETE — Remove Rows

DELETE FROM Employees
WHERE EmployeeId = 5;

-- Safe delete: check first how many rows will be affected
SELECT COUNT(*) FROM Employees WHERE Department = 'Temp';

SqlConnection

SqlConnection represents a connection to a SQL Server database. It manages the physical connection — opening, closing, and pooling. Always close connections to release resources.

Connection String Formats

C# — Connection Strings
// 1. SQL Server Authentication
string connStr1 = "Server=myServer;Database=CompanyDB;" +
                   "User Id=myUser;Password=myPass;";

// 2. Windows Authentication (Integrated Security)
string connStr2 = "Server=myServer;Database=CompanyDB;" +
                   "Integrated Security=True;";

// 3. LocalDB (development)
string connStr3 = @"Server=(localdb)\MSSQLLocalDB;Database=CompanyDB;" +
                   "Integrated Security=True;";

// 4. From appsettings.json (recommended)
// appsettings.json:
// "ConnectionStrings": {

string connStr4 = configuration.GetConnectionString("DefaultConnection");

Opening & Closing Connections

string connectionString = "Server=myServer;Database=CompanyDB;Integrated Security=True;";

{
    await connection.OpenAsync();
    // ...
}

        return false;
    }
}
Module 4

SqlCommand

SqlCommand executes SQL statements and stored procedures against a SQL Server database. It has three execution methods for different scenarios — ExecuteNonQuery, ExecuteScalar, and ExecuteReader.

Execution Methods

MethodReturnsUse Case
ExecuteNonQuery()int (rows affected)INSERT, UPDATE, DELETE
ExecuteScalar()object (first column, first row)SELECT COUNT(*), MAX(), single value
ExecuteReader()SqlDataReaderSELECT that returns multiple rows

ExecuteNonQuery — INSERT / UPDATE / DELETE

C# — ExecuteNonQuery
public int InsertEmployee(string firstName, string lastName,
                          string email, string dept, decimal salary)
{
    string sql = @"INSERT INTO Employees
                     (FirstName, LastName, Email, Department, Salary)
                   VALUES
                     (@FirstName, @LastName, @Email, @Department, @Salary)";

    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand(sql, conn);

    // Add parameters — prevents SQL injection!
    cmd.Parameters.AddWithValue("@FirstName", firstName);
    cmd.Parameters.AddWithValue("@LastName",  lastName);
    cmd.Parameters.AddWithValue("@Email",     email);
    cmd.Parameters.AddWithValue("@Department",dept);
    cmd.Parameters.AddWithValue("@Salary",   salary);

    conn.Open();
    int rowsAffected = cmd.ExecuteNonQuery();  // Returns 1 if inserted
    return rowsAffected;
}

Better Parameter Syntax — Explicit SqlParameter

C# — Typed SqlParameter
// Explicit types are preferred over AddWithValue for performance & precision
cmd.Parameters.Add(new SqlParameter("@FirstName", SqlDbType.NVarChar, 50) { Value = firstName });
cmd.Parameters.Add(new SqlParameter("@Salary",    SqlDbType.Decimal)         { Value = salary,
                                                                              Precision = 10,
                                                                              Scale = 2 });

ExecuteScalar — Single Value

C# — ExecuteScalar
// Get total employee count
public int GetEmployeeCount()
{
    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand("SELECT COUNT(*) FROM Employees", conn);

    conn.Open();
    return (int) cmd.ExecuteScalar();
}

// Get the new ID after INSERT (using OUTPUT)
public int InsertAndGetId(string firstName, string email)
{
    string sql = @"INSERT INTO Employees (FirstName, Email)
                   OUTPUT INSERTED.EmployeeId
                   VALUES (@FirstName, @Email)";

    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand(sql, conn);

    cmd.Parameters.AddWithValue("@FirstName", firstName);
    cmd.Parameters.AddWithValue("@Email", email);

    conn.Open();
    return (int) cmd.ExecuteScalar();  // Returns the new EmployeeId
}

UPDATE & DELETE Examples

C# — Update & Delete
public bool UpdateSalary(int empId, decimal newSalary)
{
    string sql = "UPDATE Employees SET Salary = @Salary WHERE EmployeeId = @Id";

    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand(sql, conn);

    cmd.Parameters.AddWithValue("@Salary", newSalary);
    cmd.Parameters.AddWithValue("@Id",     empId);

    conn.Open();
    return cmd.ExecuteNonQuery() > 0;
}

public bool DeleteEmployee(int empId)
{
    string sql = "DELETE FROM Employees WHERE EmployeeId = @Id";

    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand(sql, conn);

    cmd.Parameters.AddWithValue("@Id", empId);
    conn.Open();
    return cmd.ExecuteNonQuery() > 0;
}

Never use string concatenation for SQL! Always use parameterized queries with @paramName. String concatenation opens your app to SQL injection attacks.

Module 5

DataReader

SqlDataReader provides a fast, forward-only, read-only stream of rows from SQL Server. It's the most efficient way to read large result sets — it reads one row at a time without loading everything into memory.

Basic DataReader Usage

C# — SqlDataReader
public List<Employee> GetAllEmployees()
{
    var employees = new List<Employee>();

    string sql = "SELECT EmployeeId, FirstName, LastName, Email, Department, Salary FROM Employees";

    using var conn   = new SqlConnection(_connectionString);
    using var cmd    = new SqlCommand(sql, conn);

    conn.Open();

    using var reader = cmd.ExecuteReader();

    while (reader.Read())  // Move to next row; false when no more rows
    {
        var emp = new Employee
        {
            // Access by column name (safe, readable)
            EmployeeId = reader.GetInt32(reader.GetOrdinal("EmployeeId")),
            FirstName  = reader.GetString(reader.GetOrdinal("FirstName")),
            LastName   = reader.GetString(reader.GetOrdinal("LastName")),
            Email      = reader.GetString(reader.GetOrdinal("Email")),
            Department = reader["Department"]?.ToString(),
            Salary     = reader.GetDecimal(reader.GetOrdinal("Salary"))
        };

        employees.Add(emp);
    }

    return employees;
}

Handling NULL Values

C# — Null Handling
// Always check for NULL before reading nullable columns
while (reader.Read())
{
    int    idOrd   = reader.GetOrdinal("EmployeeId");
    int    deptOrd = reader.GetOrdinal("Department");
    int    salOrd  = reader.GetOrdinal("Salary");

    int      id         = reader.GetInt32(idOrd);
    string? department  = reader.IsDBNull(deptOrd) ? null : reader.GetString(deptOrd);
    decimal? salary     = reader.IsDBNull(salOrd)  ? (decimal?)null : reader.GetDecimal(salOrd);
}

Get by Column Index vs Name

C# — Column Access Options
// Option 1: By column index (fastest, but fragile if columns reorder)
string name = reader.GetString(1);

// Option 2: By column name (readable, recommended)
string name2 = reader["FirstName"].ToString();

// Option 3: GetOrdinal + typed getter (best: readable AND type-safe)
int firstNameOrdinal = reader.GetOrdinal("FirstName"); // cache ordinal for speed
string name3 = reader.GetString(firstNameOrdinal);

Multiple Result Sets

C# — Multiple Result Sets
string sql = "SELECT * FROM Employees; SELECT * FROM Departments;";

using var conn   = new SqlConnection(_connectionString);
using var cmd    = new SqlCommand(sql, conn);
conn.Open();

using var reader = cmd.ExecuteReader();

// Process first result set (Employees)
while (reader.Read())
    Console.WriteLine(reader["FirstName"]);

// Move to second result set (Departments)
reader.NextResult();
while (reader.Read())
    Console.WriteLine(reader["DepartmentName"]);

Async Reading

C# — Async DataReader
public async Task<List<Employee>> GetAllAsync()
{
    var list = new List<Employee>();
    await using var conn   = new SqlConnection(_connectionString);
    await using var cmd    = new SqlCommand("SELECT * FROM Employees", conn);
    await conn.OpenAsync();

    await using var reader = await cmd.ExecuteReaderAsync();
    while (await reader.ReadAsync())
        list.Add(MapEmployee(reader));

    return list;
}

DataAdapter & DataSet

SqlDataAdapter is the bridge between the database and a DataSet. It fills a DataSet with data (disconnected), and can push changes back to the database — all without keeping an open connection.

DataAdapter → DataTable

C# — Fill DataTable
public DataTable GetEmployeesTable()
{
    string sql = "SELECT * FROM Employees ORDER BY LastName";

    using var conn    = new SqlConnection(_connectionString);
    using var adapter = new SqlDataAdapter(sql, conn);

    var dt = new DataTable();
    adapter.Fill(dt);  // Opens conn, runs query, fills dt, closes conn automatically
    return dt;
}

// Use it:
DataTable table = GetEmployeesTable();

foreach (DataRow row in table.Rows)
{
    Console.WriteLine($"{row["FirstName"]} {row["LastName"]}");
    Console.WriteLine($"  Department: {row["Department"]}");
    Console.WriteLine($"  Salary: {row["Salary"]:C}");
}

DataAdapter → DataSet (Multiple Tables)

C# — Fill DataSet
public DataSet GetCompanyData()
{
    var ds = new DataSet("CompanyDS");

    using var conn = new SqlConnection(_connectionString);

    // First adapter — loads Employees table
    using var empAdapter  = new SqlDataAdapter("SELECT * FROM Employees", conn);
    empAdapter.Fill(ds, "Employees");  // Adds table named "Employees"

    // Second adapter — loads Departments table
    using var deptAdapter = new SqlDataAdapter("SELECT * FROM Departments", conn);
    deptAdapter.Fill(ds, "Departments");

    // Create a relation between the two tables
    ds.Relations.Add("Emp_Dept",
        ds.Tables["Departments"].Columns["DeptId"],
        ds.Tables["Employees"].Columns["DeptId"]);

    return ds;
}

// Navigate the relation
foreach (DataRow dept in ds.Tables["Departments"].Rows)
{
    Console.WriteLine(dept["DeptName"]);
    foreach (DataRow emp in dept.GetChildRows("Emp_Dept"))
        Console.WriteLine($"  - {emp["FirstName"]}");
}

Updating the Database via DataAdapter

C# — DataAdapter with CommandBuilder
public void UpdateWithDataAdapter()
{
    using var conn    = new SqlConnection(_connectionString);
    using var adapter = new SqlDataAdapter("SELECT * FROM Employees", conn);

    // CommandBuilder auto-generates INSERT/UPDATE/DELETE commands
    using var builder = new SqlCommandBuilder(adapter);

    var dt = new DataTable();
    adapter.Fill(dt);  // Load data

    // Modify data in memory
    DataRow newRow = dt.NewRow();
    newRow["FirstName"]  = "Frank";
    newRow["LastName"]   = "Lee";
    newRow["Email"]      = "frank@company.com";
    newRow["Department"] = "IT";
    newRow["Salary"]     = 75000;
    dt.Rows.Add(newRow);

    // Update existing row
    dt.Rows[0]["Salary"] = 90000;

    // Push ALL changes back to database in one call
    adapter.Update(dt);
}

Stored Procedures

Stored procedures are precompiled SQL code stored in the database. They provide better performance (cached execution plans), encapsulate business logic, and offer an extra security layer.

Create Stored Procedures in SQL Server

SQL — Create Stored Procedures
-- 1. Get all employees (no parameters)
CREATE PROCEDURE usp_GetAllEmployees
AS
BEGIN
    SET NOCOUNT ON;
    SELECT EmployeeId, FirstName, LastName, Email, Department, Salary
    FROM Employees
    ORDER BY LastName;
END;
GO

-- 2. Get employee by ID (input parameter)
CREATE PROCEDURE usp_GetEmployeeById
    @EmployeeId INT
AS
BEGIN
    SET NOCOUNT ON;
    SELECT * FROM Employees WHERE EmployeeId = @EmployeeId;
END;
GO

-- 3. Insert employee (with OUTPUT parameter to return new ID)
CREATE PROCEDURE usp_InsertEmployee
    @FirstName  NVARCHAR(50),
    @LastName   NVARCHAR(50),
    @Email      NVARCHAR(100),
    @Department NVARCHAR(50),
    @Salary     DECIMAL(10,2),
    @NewId      INT OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    INSERT INTO Employees (FirstName, LastName, Email, Department, Salary)
    VALUES (@FirstName, @LastName, @Email, @Department, @Salary);
    SET @NewId = SCOPE_IDENTITY();
END;
GO

-- 4. Update salary with return value (error indicator)
CREATE PROCEDURE usp_UpdateSalary
    @EmployeeId INT,
    @NewSalary  DECIMAL(10,2)
AS
BEGIN
    SET NOCOUNT ON;
    IF NOT EXISTS (SELECT 1 FROM Employees WHERE EmployeeId = @EmployeeId)
        RETURN -1;  -- Employee not found

    UPDATE Employees SET Salary = @NewSalary
    WHERE EmployeeId = @EmployeeId;
    RETURN 0;  -- Success
END;
GO

Calling Stored Procedures from C#

C# — Execute Stored Procedure
// 1. Execute stored procedure with NO parameters (returns rows)
public List<Employee> GetAll()
{
    var list = new List<Employee>();

    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand("usp_GetAllEmployees", conn);
    cmd.CommandType = CommandType.StoredProcedure;  // ← IMPORTANT!

    conn.Open();
    using var reader = cmd.ExecuteReader();
    while (reader.Read()) list.Add(MapEmployee(reader));
    return list;
}


// 2. Stored procedure with INPUT parameters
public Employee? GetById(int id)
{
    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand("usp_GetEmployeeById", conn);
    cmd.CommandType = CommandType.StoredProcedure;

    cmd.Parameters.AddWithValue("@EmployeeId", id);

    conn.Open();
    using var reader = cmd.ExecuteReader();
    return reader.Read() ? MapEmployee(reader) : null;
}


// 3. Stored procedure with OUTPUT parameter
public int InsertEmployee(Employee emp)
{
    using var conn = new SqlConnection(_connectionString);
    using var cmd  = new SqlCommand("usp_InsertEmployee", conn);
    cmd.CommandType = CommandType.StoredProcedure;

    cmd.Parameters.AddWithValue("@FirstName",  emp.FirstName);
    cmd.Parameters.AddWithValue("@LastName",   emp.LastName);
    cmd.Parameters.AddWithValue("@Email",      emp.Email);
    cmd.Parameters.AddWithValue("@Department", emp.Department);
    cmd.Parameters.AddWithValue("@Salary",     emp.Salary);

    // OUTPUT parameter — specify direction!
    var outParam = new SqlParameter("@NewId", SqlDbType.Int)
    {
        Direction = ParameterDirection.Output
    };
    cmd.Parameters.Add(outParam);

    conn.Open();
    cmd.ExecuteNonQuery();

    return (int) outParam.Value;  // The new EmployeeId
}

Always set cmd.CommandType = CommandType.StoredProcedure when calling a stored procedure. Without it, ADO.NET will try to execute the name as a SQL statement and fail.

Module 8

Transactions &
Error Handling

A transaction groups multiple database operations into a single atomic unit — either ALL succeed (Commit) or ALL are rolled back (Rollback). Essential for maintaining data integrity.

ACID Properties

A
Atomicity
All operations succeed or all are rolled back. No partial updates.
C
Consistency
Database moves from one valid state to another. Constraints are maintained.
I
Isolation
Concurrent transactions don't interfere with each other.
D
Durability
Committed transactions are permanently saved — survive crashes.

Basic Transaction Pattern

C# — SqlTransaction
public void TransferDepartment(int empId, string newDept, decimal newSalary)
{
    using var conn = new SqlConnection(_connectionString);
    conn.Open();

    // Begin transaction on the open connection
    using SqlTransaction transaction = conn.BeginTransaction();

    try
    {
        // Command 1: Update department
        using var cmd1 = new SqlCommand(
            "UPDATE Employees SET Department = @Dept WHERE EmployeeId = @Id", conn, transaction);
        cmd1.Parameters.AddWithValue("@Dept", newDept);
        cmd1.Parameters.AddWithValue("@Id",   empId);
        cmd1.ExecuteNonQuery();

        // Command 2: Update salary
        using var cmd2 = new SqlCommand(
            "UPDATE Employees SET Salary = @Salary WHERE EmployeeId = @Id", conn, transaction);
        cmd2.Parameters.AddWithValue("@Salary", newSalary);
        cmd2.Parameters.AddWithValue("@Id",     empId);
        cmd2.ExecuteNonQuery();

        // All succeeded — commit!
        transaction.Commit();
        Console.WriteLine("Transaction committed.");
    }
    catch (SqlException ex)
    {
        // Something went wrong — roll back all changes
        transaction.Rollback();
        Console.WriteLine($"Transaction rolled back: {ex.Message}");
        throw;  // Re-throw so caller knows it failed
    }
}

Nested Transactions with Savepoints

C# — Savepoints
using SqlTransaction tx = conn.BeginTransaction();
try
{
    // First batch of operations
    ExecuteInsert(conn, tx, emp1);
    ExecuteInsert(conn, tx, emp2);

    // Create a savepoint after successful first batch
    tx.Save("Savepoint1");

    try
    {
        // Second batch (might fail)
        ExecuteInsert(conn, tx, emp3);
    }
    catch
    {
        // Only roll back to savepoint, keep first batch
        tx.Rollback("Savepoint1");
    }

    tx.Commit();  // Commit what survived
}
catch
{
    tx.Rollback();  // Roll back everything
    throw;
}

Error Handling — SqlException

C# — Comprehensive Error Handling
public bool SafeInsert(Employee emp)
{
    try
    {
        using var conn = new SqlConnection(_connectionString);
        using var cmd  = new SqlCommand("usp_InsertEmployee", conn);
        cmd.CommandType  = CommandType.StoredProcedure;
        cmd.CommandTimeout = 30;  // seconds before timeout
        // ... add parameters
        conn.Open();
        cmd.ExecuteNonQuery();
        return true;
    }
    catch (SqlException ex) when (ex.Number == 2627)
    {
        // Error 2627 = unique constraint violation
        Console.WriteLine($"Duplicate email: {emp.Email}");
        return false;
    }
    catch (SqlException ex) when (ex.Number == -2)
    {
        // Error -2 = timeout expired
        Console.WriteLine("Query timed out.");
        throw;
    }
    catch (SqlException ex)
    {
        // Log all SqlException details
        foreach (SqlError error in ex.Errors)
        {
            Console.WriteLine($"  Error #{error.Number}: {error.Message}");
            Console.WriteLine($"  Server: {error.Server}, Proc: {error.Procedure}");
        }
        throw;
    }
    catch (InvalidOperationException ex)
    {
        // Connection not opened, or already open, etc.
        Console.WriteLine($"Connection error: {ex.Message}");
        throw;
    }
}

Repository — Data Access Layer

C# — Data/EmployeeRepository.cs
using System.Data;
using Microsoft.Data.SqlClient;
using EmployeeManager.Models;

namespace EmployeeManager.Data;

public class EmployeeRepository
{
    private readonly string _connStr;

    public EmployeeRepository(string connectionString) =>
        _connStr = connectionString;

    // ── HELPER: Map DataReader row → Employee ──────────────
    private static Employee Map(SqlDataReader r) => new()
    {
        EmployeeId = r.GetInt32(r.GetOrdinal("EmployeeId")),
        FirstName  = r.GetString(r.GetOrdinal("FirstName")),
        LastName   = r.GetString(r.GetOrdinal("LastName")),
        Email      = r.GetString(r.GetOrdinal("Email")),
        Department = r.IsDBNull(r.GetOrdinal("Department")) ? null
                     : r.GetString(r.GetOrdinal("Department")),
        Salary     = r.GetDecimal(r.GetOrdinal("Salary")),
        HireDate   = r.GetDateTime(r.GetOrdinal("HireDate"))
    };

    // ── READ: Get all employees ─────────────────────────────
    public List<Employee> GetAll()
    {
        var list = new List<Employee>();
        using var conn = new SqlConnection(_connStr);
        using var cmd  = new SqlCommand("usp_GetAllEmployees", conn)
                        { CommandType = CommandType.StoredProcedure };
        conn.Open();
        using var r = cmd.ExecuteReader();
        while (r.Read()) list.Add(Map(r));
        return list;
    }

    // ── READ: Get by ID ─────────────────────────────────────
    public Employee? GetById(int id)
    {
        using var conn = new SqlConnection(_connStr);
        using var cmd  = new SqlCommand("usp_GetEmployeeById", conn)
                        { CommandType = CommandType.StoredProcedure };
        cmd.Parameters.AddWithValue("@EmployeeId", id);
        conn.Open();
        using var r = cmd.ExecuteReader();
        return r.Read() ? Map(r) : null;
    }

    // ── CREATE ──────────────────────────────────────────────
    public int Insert(Employee emp)
    {
        using var conn = new SqlConnection(_connStr);
        using var cmd  = new SqlCommand("usp_InsertEmployee", conn)
                        { CommandType = CommandType.StoredProcedure };

        cmd.Parameters.AddWithValue("@FirstName",  emp.FirstName);
        cmd.Parameters.AddWithValue("@LastName",   emp.LastName);
        cmd.Parameters.AddWithValue("@Email",      emp.Email);
        cmd.Parameters.AddWithValue("@Department", emp.Department ?? (object)DBNull.Value);
        cmd.Parameters.AddWithValue("@Salary",     emp.Salary);

        var newIdParam = new SqlParameter("@NewId", SqlDbType.Int)
                        { Direction = ParameterDirection.Output };
        cmd.Parameters.Add(newIdParam);

        conn.Open();
        cmd.ExecuteNonQuery();
        return (int) newIdParam.Value;
    }

    // ── UPDATE ──────────────────────────────────────────────
    public bool Update(Employee emp)
    {
        const string sql = @"UPDATE Employees
                            SET FirstName=@FN, LastName=@LN,
                                Email=@Em, Department=@Dp, Salary=@Sl
                            WHERE EmployeeId=@Id";

        using var conn = new SqlConnection(_connStr);
        using var cmd  = new SqlCommand(sql, conn);

        cmd.Parameters.AddWithValue("@FN", emp.FirstName);
        cmd.Parameters.AddWithValue("@LN", emp.LastName);
        cmd.Parameters.AddWithValue("@Em", emp.Email);
        cmd.Parameters.AddWithValue("@Dp", emp.Department ?? (object)DBNull.Value);
        cmd.Parameters.AddWithValue("@Sl", emp.Salary);
        cmd.Parameters.AddWithValue("@Id", emp.EmployeeId);

        conn.Open();
        return cmd.ExecuteNonQuery() > 0;
    }

    // ── DELETE (with transaction) ───────────────────────────
    public bool Delete(int id)
    {
        using var conn = new SqlConnection(_connStr);
        conn.Open();
        using var tx = conn.BeginTransaction();
        try
        {
            // Log the deletion first
            using var log = new SqlCommand(
                "INSERT INTO AuditLog (Action, TableName, RecordId, ActionDate) VALUES ('DELETE','Employees',@Id,GETDATE())",
                conn, tx);
            log.Parameters.AddWithValue("@Id", id);
            log.ExecuteNonQuery();

            // Then delete
            using var del = new SqlCommand(
                "DELETE FROM Employees WHERE EmployeeId = @Id", conn, tx);
            del.Parameters.AddWithValue("@Id", id);
            int rows = del.ExecuteNonQuery();

            tx.Commit();
            return rows > 0;
        }
        catch
        {
            tx.Rollback();
            throw;
        }
    }
}

using EmployeeManager.Data;
using EmployeeManager.Models;

namespace EmployeeManager.UI;

public class ConsoleMenu(EmployeeRepository repo)
{
    public void Run()
    {
        while (true)
        {
            Console.WriteLine("\n╔══ Employee Manager ══╗");
            Console.WriteLine("║ 1. List All          ║");
            Console.WriteLine("║ 2. Find by ID        ║");
            Console.WriteLine("║ 3. Add Employee      ║");
            Console.WriteLine("║ 4. Update Employee   ║");
            Console.WriteLine("║ 5. Delete Employee   ║");
            Console.WriteLine("║ 0. Exit              ║");
            Console.WriteLine("╚══════════════════════╝");
            Console.Write("Choice: ");

            switch (Console.ReadLine())
            {
                case "1": ListAll();    break;
                case "2": FindById();  break;
                case "3": AddNew();    break;
                case "4": UpdateEmp(); break;
                case "5": DeleteEmp(); break;
                case "0": return;
                default: Console.WriteLine("Invalid choice."); break;
            }
        }
    }

    private void ListAll()
    {
        var emps = repo.GetAll();
        if (!emps.Any()) { Console.WriteLine("No employees found."); return; }
        Console.WriteLine($"\n{"ID",-5} {"Name",-22} {"Department",-16} {"Salary",10}");
        Console.WriteLine(new string('-', 55));
        foreach (var e in emps)
            Console.WriteLine($"{e.EmployeeId,-5} {e.FirstName+" "+e.LastName,-22} {e.Department,-16} {e.Salary,10:C}");
    }

    private void FindById()
    {
        Console.Write("Enter ID: ");
        if (!int.TryParse(Console.ReadLine(), out int id)) return;
        var emp = repo.GetById(id);
        Console.WriteLine(emp is null ? "Not found." : emp.ToString());
    }

    private void AddNew()
    {
        var emp = new Employee();
        Console.Write("First Name: ");  emp.FirstName  = Console.ReadLine()!;
        Console.Write("Last Name: ");   emp.LastName   = Console.ReadLine()!;
        Console.Write("Email: ");       emp.Email      = Console.ReadLine()!;
        Console.Write("Department: "); emp.Department = Console.ReadLine();
        Console.Write("Salary: ");
        if (!decimal.TryParse(Console.ReadLine(), out decimal sal)) return;
        emp.Salary = sal;

        try
        {
            int newId = repo.Insert(emp);
            Console.WriteLine($"✓ Employee added with ID: {newId}");
        }
        catch (SqlException ex) when (ex.Number == 2627)
        {
            Console.WriteLine("✗ Error: Email already exists.");
        }
    }

    private void UpdateEmp()
    {
        Console.Write("Employee ID to update: ");
        if (!int.TryParse(Console.ReadLine(), out int id)) return;
        var emp = repo.GetById(id);
        if (emp is null) { Console.WriteLine("Not found."); return; }

        Console.Write($"New Salary [{emp.Salary:C}]: ");
        if (decimal.TryParse(Console.ReadLine(), out decimal sal))
            emp.Salary = sal;

        Console.WriteLine(repo.Update(emp) ? "✓ Updated." : "✗ Update failed.");
    }

    private void DeleteEmp()
    {
        Console.Write("Employee ID to delete: ");
        if (!int.TryParse(Console.ReadLine(), out int id)) return;
        Console.Write("Are you sure? (y/n): ");
        if (Console.ReadLine()?.ToLower() != "y") return;
        Console.WriteLine(repo.Delete(id) ? "✓ Deleted." : "✗ Not found.");
    }
}

Key Concepts Used in the Project

1
Repository Pattern
Data access logic is separated into EmployeeRepository. The UI doesn't know about SQL — it just calls methods. Clean, testable, maintainable.
2
Parameterized Queries
Every SQL statement uses @paramName syntax — never string concatenation. This prevents SQL injection completely.
3
using Statements
Every SqlConnection, SqlCommand, and SqlDataReader is wrapped in using to guarantee disposal and connection pool return.
4
Stored Procedures for Read Operations
GetAll and GetById use stored procedures (better performance with cached execution plans). Update uses inline SQL to show both approaches.
5
Transaction on Delete
Delete writes an audit log entry AND deletes the record in a single transaction. If the audit log fails, the delete is rolled back — maintaining integrity.
6
Configuration via appsettings.json
Connection string lives in configuration, not hardcoded. Change the environment (dev/prod) by swapping config — code doesn't change.
